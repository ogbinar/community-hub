Python Fundamentals

Learn to interrogate AI-generated Python code — spot common bugs in data manipulation, understand why AI sometimes hallucinates pandas methods, and practice validating outputs.

Concept Explanation

Python is the most popular programming language for data work. The key library you'll use is pandas — it's like Excel on steroids, but automated and reproducible.

Three Core pandas Operations

  • Reading data — Load a CSV or Excel file into a DataFrame (like opening a spreadsheet)
  • Filtering & transforming — Select rows, create new columns, clean messy data
  • Aggregating — Group by category and calculate sums, averages, counts

The basic pattern for reading a file is:

import pandas as pd
df = pd.read_csv('data.csv')
print(df.head())

AI-Generated Answer (What ChatGPT/Copilot Would Output)

Here's what an AI would generate if asked: "Write Python code to calculate the average salary by department from a DataFrame."

# AI-generated code
import pandas as pd

df = pd.read_csv('employees.csv')
avg_salary_by_dept = df.groupby('department')['salary'].mean()
print(avg_salary_by_dept)

The code looks correct at first glance. But does it handle edge cases? Let's investigate.

Where AI Gets It Wrong

This code has three common pitfalls that AI often misses:

  1. 1
    No handling of missing values. If the salary column has NaN values, .mean() will skip them silently — but you won't know how many were missing. Always check with .isna().sum() first.
  2. 2
    No handling of empty departments. If a department has zero employees or all NaN salaries, the result will show NaN — which can break downstream code.
  3. 3
    No type checking. If 'salary' is stored as a string (common with messy CSV files), .mean() will fail with a TypeError. AI rarely warns about this.
# Safer version
import pandas as pd

df = pd.read_csv('employees.csv')

# Check for issues first
print(f"Missing salaries: {df['salary'].isna().sum()}")
print(f"Data types:\n{df.dtypes}")

# Convert salary to numeric, coercing errors to NaN
df['salary'] = pd.to_numeric(df['salary'], errors='coerce')

# Calculate with proper handling
avg_salary_by_dept = df.groupby('department')['salary'].mean().round(2)
print(avg_salary_by_dept)

How a Practitioner Validates This Code

Before trusting any AI-generated Python code, practitioners follow these validation steps:

  1. 1
    Check data types. Run df.dtypes to ensure columns are the right type before operations.
  • 2
    Check for missing values. Run df.isna().sum() to see how many NaN values exist in each column.
  • 3
    Test with sample data. Create a small DataFrame (5–10 rows) and verify the output matches expectations.
  • 4
    Check edge cases. What happens if the column is empty? If all values are NaN? If there's a single row?
  • 5
    Run in a notebook. Execute cell-by-cell to catch errors early, not at the end.
