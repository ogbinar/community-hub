Step-by-Step Gap Closer

  1. Understand star schema — Learn why fact tables and dimension tables matter.
  2. Create relationships in Power BI between your tables (one-to-many, many-to-one).
  3. Avoid bi-directional filters unless you know what you're doing.
  4. Use the Model View to verify relationships are correct.

Data Modeling Checklist

- [ ] Fact table has numeric measures (Sales, Quantity, Amount)
- [ ] Dimension tables have descriptive columns (Customer, Product, Date)
- [ ] Relationships are one-to-many from dimension to fact
- [ ] No circular relationships exist
- [ ] Date table is marked as a date table in Power BI
  1. Learn the difference between measures (aggregated on the fly) and calculated columns (stored in memory).
  2. Write basic DAX functions: SUM, AVERAGE, COUNT, DISTINCTCOUNT.
  3. Use CALCULATE — the most important DAX function for filtering context.

Essential DAX Functions

Total Sales = SUM(Sales[Amount])

Total YTD = TOTALYTD([Total Sales], 'Date'[Date])

Sales vs Prior Year = 
CALCULATE(
  [Total Sales],
  SAMEPERIODLASTYEAR('Date'[Date])
)

% of Total = DIVIDE([Total Sales], ALLSELECTED([Total Sales]))
  1. Use a consistent color scheme — pick 2–3 colors max.
  2. Place KPIs at the top, charts below, details at the bottom.
  3. Avoid clutter — white space is your friend.
  4. Add tooltips for context on hover.
