How to Contribute via GitHub

  1. Fork the repository: Click "Fork" on any DEP project repo
  2. Create a branch: Use descriptive names like fix/typo-in-readme
  3. Make changes: Edit files and commit with clear messages
  4. Create a Pull Request (PR): Open a PR describing your changes

PR Template

## What I Changed
Brief description of the changes made.

## Why I Changed It
The reason for this change (e.g., fixing a bug, adding documentation).

## How to Test
Steps to verify the changes work correctly.

Code Review Best Practices

  • Be specific: Point to exact lines and explain what needs improvement
  • Be constructive: Suggest improvements, not just criticism
  • Be timely: Review PRs within 24–48 hours when possible

Discord Channels for Contributors

  • #code-reviews: Post code snippets for community review
  • #documentation: Discuss documentation improvements and templates
  • #content-creation: Share YouTube scripts, blog posts, and tutorials
  • #project-management: Track project progress and assign tasks

Discord Etiquette

• Use threads for specific discussions to keep channels organized
• Tag @mentor when you need help from a volunteer mentor
• Be patient — async means responses may take hours, not minutes
• Share your work-in-progress in #showcase for feedback

Collaborative Documentation

  1. Create a Google Doc: Start with a clear title and description
  2. Share with collaborators: Add email addresses or use shareable links
  3. Use comments: Leave feedback on specific sections using the comment feature
  4. Track changes: Google Docs automatically tracks who edited what and when

Documentation Templates

  • Project README: Overview, setup instructions, usage examples
  • API Documentation: Endpoint descriptions, request/response formats
  • Tutorial Guide: Step-by-step instructions with screenshots

Async Workflows for Overseas Contributors

Timezone-Aware Collaboration

  1. Post updates in Discord threads: Use #async-updates to share progress across timezones
  2. Leave detailed comments: When reviewing PRs or docs, explain your reasoning so others understand your perspective
  3. Use async meetings: Record 5-minute Loom videos explaining complex topics instead of scheduling live calls
  4. Rotate meeting times: If you need a live call, rotate the time to accommodate different timezones
