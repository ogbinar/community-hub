Concept Explanation

Python is the most popular language for data work. The pandas library is like Excel inside Python — it lets you load, filter, transform, and analyze datasets with code.

The Three Pandas Objects

  • Series — A single column of data (like one Excel column)
  • DataFrame — Multiple columns organized in rows and columns (like an entire Excel sheet)
  • Index — The row labels that let you identify specific rows

The most common workflow is: load data → filter/transform → analyze → export. Here's the basic pattern:

# Load data, filter, and analyze
import pandas as pd

# Load a CSV file
df = pd.read_csv('sales_data.csv')

# Filter rows where amount > 5000
filtered = df[df['amount'] > 5000]

# Group by region and sum amounts
result = filtered.groupby('region')['amount'].sum()

AI-Generated Answer (What ChatGPT/Copilot Would Output)

Here's what an AI would generate if asked: "Write Python code to calculate the average sale per region from a sales DataFrame."

# AI-generated code
import pandas as pd

df = pd.read_csv('sales.csv')
avg_by_region = df.groupby('region')['amount'].mean()
print(avg_by_region)

The code looks simple and correct. But is it? Let's find out where it might be wrong.

Where AI Gets It Wrong

This code has three subtle bugs that AI commonly introduces:

  1. 1
    Bug: No handling of missing values. If the dataset has NaN or None values, mean() will produce incorrect results. A practitioner would add .dropna() or .fillna(0) first.
  2. 2
    Bug: No error handling for missing columns. If the CSV doesn't have a 'region' column, the code crashes silently with a KeyError. Best practice wraps this in try/except or checks df.columns first.
  3. 3
    Bug: No data type validation. If 'amount' is stored as strings instead of numbers, mean() will fail or produce wrong results. A practitioner would use pd.to_numeric() to ensure correct types.
# Fixed version (with validation)
import pandas as pd

df = pd.read_csv('sales.csv')

# Ensure correct data types
df['amount'] = pd.to_numeric(df['amount'], errors='coerce')

# Drop rows with missing values before calculation
df_valid = df.dropna(subset=['region', 'amount'])

# Calculate average per region
avg_by_region = df_valid.groupby('region')['amount'].mean()

print(avg_by_region)

Key takeaway: Always validate data types, handle missing values, and check column names before trusting AI-generated pandas code.

Practitioner Validation Checklist

Before trusting any AI-generated Python data code, practitioners follow these validation steps:

  1. 1
    Check data types. Run df.dtypes to verify columns are the right type (numeric, string, datetime).
  • 2
    Count missing values. Run df.isnull().sum() to see how many NaN values exist in each column.
  • 3
    Verify column names. Print df.columns to confirm the exact spelling of columns before referencing them.
  • 4
    Check result shape. After groupby, verify the number of groups matches your expectations with .nunique().
  • 5
    Test with sample data. Create a small DataFrame with known values and compare output against expected results.
