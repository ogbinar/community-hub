Concept Explanation

SQL (Structured Query Language) is the language you use to talk to databases. Think of it like asking a question — you tell the database what data you want, and it gives you back exactly that.

The Four Core Operations

  • SELECT — Get data from a table (like asking "show me all customers")
  • INSERT — Add new data (like "add this new customer")
  • UPDATE — Change existing data (like "change this customer's email")
  • DELETE — Remove data (like "remove this customer")

For this module, we'll focus on SELECT — it's the most common operation and the one you'll use 90% of the time. The basic pattern is:

-- Basic SELECT query
SELECT column1, column2
FROM table_name
WHERE condition = value;

Try it yourself: Use SQLiteOnline.com (free, no install needed) to run the query above with your own data.

AI-Generated Answer (What ChatGPT/Copilot Would Output)

Here's what an AI would generate if asked: "Write a SQL query to find all customers who spent more than ₱5000 in 2024."

-- AI-generated query
SELECT customer_id, customer_name, SUM(amount) as total_spent
FROM customers c
JOIN orders o ON c.id = o.customer_id
WHERE YEAR(order_date) = 2024
GROUP BY customer_id, customer_name
HAVING SUM(amount) > 5000;

The AI provided a query that looks correct. But is it? Let's find out where it might be wrong.

Where AI Gets It Wrong

This query has three subtle bugs that AI commonly introduces:

  1. 1
    Bug: YEAR() is not standard SQL. It works in MySQL and SQL Server, but NOT in PostgreSQL or SQLite. A portable version would use date range comparison instead:
  2. 2
    Bug: Missing alias for the JOIN. If both tables have an "id" column, the query will fail with an "ambiguous column" error unless you explicitly reference c.id.
  3. 3
    Bug: INNER JOIN excludes customers with no orders. If you want ALL customers (even those who spent less than ₱5000 or have zero orders), use LEFT JOIN and filter after aggregation.
-- Fixed version (works across all databases)
SELECT c.customer_id, c.customer_name, SUM(o.amount) as total_spent
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id
WHERE o.order_date >= '2024-01-01' AND o.order_date < '2025-01-01'
GROUP BY c.customer_id, c.customer_name
HAVING SUM(o.amount) > 5000;

Key takeaway: Always verify AI output against your specific database's documentation. Functions like YEAR(), DATE_FORMAT(), and GETDATE() are not portable.

Practitioner Validation Checklist

Before trusting any AI-generated SQL, practitioners follow these validation steps:

  1. 1
    Check database compatibility. Does the query use functions specific to one database? (YEAR(), DATE_FORMAT(), etc.)
  • 2
    Test with a small dataset. Run the query against 10–20 rows first. Check if results match your expectations.
  • 3
    Verify JOIN logic. Is it INNER or LEFT? Will you lose rows? Check the join condition for duplicates.
  • 4
    Check GROUP BY matches SELECT. Every non-aggregated column in SELECT must appear in GROUP BY (SQL standard).
  • 5
    Run EXPLAIN or execution plan. Is the query efficient? Are indexes being used?
