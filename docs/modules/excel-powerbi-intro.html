Concept Explanation

Excel and Google Sheets look similar, but they use different formulas. AI often confuses them, giving you the wrong formula for your platform.

VLOOKUP vs INDEX/MATCH

  • VLOOKUP — Simple, but limited. Searches only left-to-right and can't find the first column.
  • INDEX/MATCH — More flexible. Works in both directions and is faster on large datasets.

Here's how VLOOKUP works in Excel:

-- Excel formula
=VLOOKUP(A2, D:F, 3, FALSE)

-- INDEX/MATCH (more flexible)
=INDEX(C:C, MATCH(A2, B:B, 0))

Try it yourself: Open Excel or Google Sheets and experiment with VLOOKUP using a sample dataset. Compare the results with INDEX/MATCH.

AI-Generated Answer (What ChatGPT/Copilot Would Output)

Here's what an AI would generate if asked: "Write a DAX measure to calculate year-over-year sales growth in Power BI."

-- AI-generated DAX measure
YoY Growth = (SUM(Sales[Amount]) - SUM(PreviousYear[Amount])) / SUM(PreviousYear[Amount])

The formula looks straightforward. But is it correct for Power BI? Let's find out where it might be wrong.

Where AI Gets It Wrong

This DAX code has three subtle bugs that AI commonly introduces:

  1. 1
    Bug: No CALCULATE wrapper. In Power BI, aggregation functions need to be wrapped in CALCULATE() to respect filter context. Without it, the measure ignores date filters.
  2. 2
    Bug: No handling of division by zero. If previous year sales are zero or null, the formula returns BLANK() or an error. A practitioner would use DIVIDE() instead.
  3. 3
    Bug: No proper date table relationship. Time intelligence functions (SAMEPERIODLASTYEAR, PARALLELPERIOD) require a marked date table. Without it, the calculation fails silently.
-- Fixed version (proper DAX)
YoY Growth = 
VAR CurrentSales = CALCULATE(SUM(Sales[Amount]))
VAR PreviousSales = CALCULATE(
    SUM(Sales[Amount]),
    SAMEPERIODLASTYEAR('Date'[Date])
)
RETURN
DIVIDE(CurrentSales - PreviousSales, PreviousSales)

Key takeaway: DAX is not Excel. AI often generates Excel formulas when asked for DAX. Always check if the function exists in DAX (e.g., DIVIDE() instead of /, CALCULATE() wrappers).

Practitioner Validation Checklist

Before trusting any AI-generated Excel or DAX code, practitioners follow these validation steps:

  1. 1
    Check platform compatibility. Is this formula for Excel, Google Sheets, or Power BI? They use different function names and syntax.
  • 2
    Verify function existence. Does the function exist in your platform? (e.g., VLOOKUP works in Excel but not Google Sheets)
  • 3
    Test with known values. Enter simple numbers and check if the result matches your manual calculation.
  • 4
    Check for division by zero. Use IFERROR() in Excel or DIVIDE() in DAX to handle edge cases gracefully.
  • 5
    Validate date relationships. In Power BI, verify that your Date table is marked and linked to fact tables before using time intelligence functions.
