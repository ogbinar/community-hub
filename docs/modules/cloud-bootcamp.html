AWS Free Tier Setup

Step 1: Create an AWS Account

  1. Go to aws.amazon.com/free
  2. Click "Create a Free Account" and follow the sign-up process
  3. Verify your phone number (required for security)
  4. Add a payment method (you won't be charged within free tier limits)

Step 2: Set Up EC2 (Virtual Machine)

  1. Log in to the AWS Console and search for "EC2"
  2. Click "Launch Instance" and choose Amazon Linux 2023
  3. Select t2.micro (free tier eligible)
  4. Create a key pair (download the .pem file — keep it safe!)
  5. Launch the instance and note the public IP address

Step 3: Connect to Your VM

SSH Connection Command

# On your local machine terminal:
chmod 400 your-key-pair.pem
ssh -i "your-key-pair.pem" ec2-user@YOUR_PUBLIC_IP

# Once connected, install Python and pandas:
sudo yum update -y
sudo yum install python3 -y
pip3 install pandas openpyxl

Step 4: Deploy Your First Script

Deploy Script to EC2

# From your local machine:
scp -i "your-key-pair.pem" analysis.py ec2-user@YOUR_PUBLIC_IP:/home/ec2-user/

# SSH into the VM and run:
ssh -i "your-key-pair.pem" ec2-user@YOUR_PUBLIC_IP
python3 analysis.py

Azure Free Tier Setup

Step 1: Create an Azure Account

  1. Go to azure.microsoft.com/free
  2. Click "Start free" and sign up with a Microsoft account
  3. You get ₱2,300 in free credits for 12 months

Step 2: Set Up Azure VM

  1. Log in to the Azure Portal and search for "Virtual Machines"
  2. Create a new VM with Ubuntu Server 22.04
  3. Select B1s instance (free tier eligible)
  4. Set up SSH key authentication

Step 3: Deploy Python Script

Azure VM Setup Commands

# SSH into your Azure VM:
ssh -i "your-key.pem" azureuser@YOUR_PUBLIC_IP

# Install Python and pandas:
sudo apt update
sudo apt install python3-pip -y
pip3 install pandas openpyxl

# Run your script:
python3 analysis.py

Step 4: Set Up Monitoring

  1. Resource Groups: Organize your resources into groups for easy management
  2. Azure Monitor: Set up basic monitoring to track CPU, memory, and disk usage
  3. Alerts: Create alerts when CPU exceeds 80% for 5 minutes

CI/CD Pipeline with GitHub Actions

Automate Your Deployment

  1. Create a GitHub repository for your project
  2. Add a .github/workflows/ directory to your repo
  3. Create a CI/CD workflow file (see example below)
  4. Push changes to GitHub — the pipeline runs automatically
